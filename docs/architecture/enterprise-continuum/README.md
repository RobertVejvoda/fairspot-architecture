This section is a learning guide for applying TOGAF Enterprise Continuum thinking to FairSpot.

The worked example is a fictive company, **FairSpot Operator**, that runs FairSpot commercially. The purpose is to learn how reusable architecture and solution assets can be classified, reused, specialized, and governed when moving from a product architecture to an enterprise architecture.

## Boundary

FairSpot product architecture remains under the existing Business, Information Systems, Technology, Security, Governance, and Implementation sections.

FairSpot Operator architecture describes the fictive enterprise that commercializes, deploys, supports, and governs FairSpot. It reuses product architecture, but it adds enterprise concerns such as customer acquisition, onboarding, support, operating model, commercial service packages, customer evidence, and platform operation.
