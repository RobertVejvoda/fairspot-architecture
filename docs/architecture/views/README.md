# Views And Diagrams

Architecture views and diagrams answer stakeholder concerns across the FairSpot target architecture.

The existing diagram exports under `docs/images/`, `docs/archi/`, and draw.io files remain source evidence until the target view catalog marks a replacement view as authoritative. Source-evidence diagrams can explain history and earlier design intent, but they should not override the current target architecture pages when there is a conflict.

## Contents

- [Viewpoint Catalog](/architecture/views/viewpoint-catalog)
- [Diagrams](/architecture/views/diagrams)

## Diagram Governance

- Each target diagram must be tied to a viewpoint and a stakeholder concern.
- Each source-evidence diagram must say whether it can be reused, annotated, or replaced.
- Missing diagrams stay visible as placeholders until Robert supplies or approves the refreshed ArchiMate/draw.io view.
- A diagram becomes authoritative only after it is indexed in [Diagrams](/architecture/views/diagrams), linked to the relevant architecture artifact, and reviewed with the artifact it supports.
